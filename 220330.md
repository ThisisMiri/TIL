1. 지금까지 공부한 HTML 태그들 처음부터 쭉 살펴보기      

    ```   
    [Text Level Semantic]   
    텍스트를 쓸 때 사용하는 태그들. 공간을 어떻게 차지하느냐가 그룹핑과 텍스트레벨의 차이점.  

    
    - <br> 줄바꿈을 위한 태그.
      <wbr> 근래에는 사용 잘 안함. 반응형 디자인에 사용. 의미없는 단어나 해석할 수 없는 단어의 줄바꿈할 때 사용.
    💡 white space = 공백    

    - <a href="경로"> 링크를 만들 때 사용.
    target="blank" : 새 창으로 링크 이동.
    <a href="#경로"> 페이지 내에서 이동시 사용. 목차에서 주로 사용.
    <a href="다운로드 파일" download="다운로드 파일 이름명"> 다운로드 할 파일과 파일의 이름명까지 지정 가능.   

    - <b> 굵은 글씨 표현. 의미 X. CSS 등장 이전의 태그.
      <strong> 굵은 글씨 표현 + 중요도. 중첩사용 불가. 
    둘의 디자인적 차이 없고 강조의 차이.

    -<i> 의미 없고, 기울림 글꼴 표현. CSS 등장 이전의 태그.
    <em> 기울림 글꼴 표현, 강조의 의미 갖음.

    -<dfn> 문맥에서 정의하는 내용.

    -<abbr> 약자를 나타내고 싶을 때 사용. 보통 혼자 사용.
    
    -<sup> 윗첨자 
     <sub> 아랫첨자

    -<span> 줄바꿈없이 묶음. 의미는 없음. div처럼 최후의 수단으로 사용.
    💡어쩔 수 없이 사용해야 한다면, 글씨묶기 -> span
                                    요소묶기 -> div


    [Embedded Content]
    - <img> 이미지 삽입 태그. src는 필수속성.
      "큰 따옴표"안의 경로는 절대 경로와 상대 경로, 두 가지.
           절대경로 : original, / 사용.
           상대경로 : ./ = 현재 거기 그 자리 (폴더 안)
                      ../ = 폴더 밖   

    alt : 대체 텍스트 = 이미지 설명, 다 넣을 필요는 없으나 비워서라도 두기.
          SEO와 스크린 리더에 영향을 미침!

    srcset : 각기 다른 환경에서 사용될 이미지 소스 명시. 🚫 IE 지원X
        x서술자 : 화소의 밀도를 나타내다.
        y서술자 : 원본 이미지의 넓이가 차례대로 300px, 600px, 700px 임을 브라우저에게 알려줌. 표기는 px가 아닌 w.
        sizes 속성 :  뷰포트의 조건에 따라 이미지가 UI안에서 차지하게 될 사이즈를 브라우저에 알려줌.   

    <picture> : <source> <img>를 통해 각자 다른 디스플레이, 혹은 디바이스에 따라 조건에 맞는 이미지를 보여줌.

    -media 속성
    -type 속성
    
    ⭐️ 점진적 향상기법(면접에 나올 수 있어요!)
    기본적으로 예전 기술 환경에서 동작할 수 있는 기능을 구현해두고, 최신 기술을 사용할 수 있는 환경에서는 최신 기술을 제공하여 더 나은 사용자 경험을 제공할 수 있는 방법.

    💡 이미지포맷 종류 (어느정도 알아두기!) 📷
    GIF : 256색의 컬러만 표현 가능. 그렇기에 선명하진 않지만 용량이 적음. 투명은 표현 가능하지만 그 정도를 조절하는건 불가능. 때문에 그림자 표현도 불가능. 애니메이션 처리 가능.

    JPG/JPEG : 매우 화소가 높고, 용량도 적지만 투명처리가 불가능

    PNG : 왠만한 컬러는 모두 표현 가능. 투명 영역을 처리 가능. 용량이 큼.

    SVG : SVG 형식의 벡터 이미지는 손실이나 품질 저하 없이 모든 크기에서 렌더링이 가능. 다만 이미지 모양이 복잡해지면 코드가 많아지고, 용량도 늘어남.

    WEBP : 압축률이 좋다고 소문난 JPEG 이미지에 비해 무려 용량은 70% 수준으로 낮지만 더 뛰어난 색상을 지원하는 포맷. PNG 처럼 투명도 표현 가능. GIF 처럼 애니메이션 표현도 가능. 만능 포맷!

    AVIF : WebP 처럼 뛰어난 색상표현, 애니메이션 지원, 투명도 표현 모두 가능. JPEG 이미지의 50% 수준인 용량을 자랑하는 차세대 이미지 포맷. 아직 지원하지 않는 브라우저가 많음.
